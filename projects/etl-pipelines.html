Context

As part of Bank SinoPac's digital banking data team, I was responsible for designing and maintaining production ETL pipelines that fed dashboards, downstream models, and executive reporting. The stakes were real: bad data meant bad decisions in a regulated financial environment.

Pipeline Design

The core pipeline processed 5M+ daily records using Tableau Prep, KNIME, and SSIS — each tool chosen based on the specific transformation needs of the data source. Manual data preparation that had previously taken 8 hours per cycle was reduced to 15 minutes through workflow automation.

Schema inconsistencies from upstream banking systems were handled via standardization layers before data entered the warehouse, ensuring consistent field definitions across reporting surfaces.

Data Quality Monitoring

Pipelines without monitoring are a liability. I built a real-time data quality framework using Postman and MongoDB Compass that implemented automated validation rules at ingestion — checking for null rates, value distributions, referential integrity, and statistical anomalies.

When a validation rule fired, the framework surfaced the incident immediately rather than letting bad data propagate silently into dashboards. This reduced data incidents by 40% and gave the team confidence to ship faster.

Database Optimization

Query performance on legacy MSSQL tables with 100M+ rows had degraded to the point where some reports timed out. Through strategic indexing, query rewrites, and partition pruning, average query time dropped from 45 seconds to 3 seconds — enabling real-time analytics that weren't previously feasible.
