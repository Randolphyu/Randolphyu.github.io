The Problem

At Bank SinoPac, business teams needed data — but getting it meant filing a request, waiting for an analyst, and hoping the output matched what they actually wanted. The SQL bottleneck was real: non-technical stakeholders couldn't self-serve, and analysts spent a disproportionate amount of time on ad-hoc queries instead of higher-leverage work.

The question I set out to answer: could an LLM reliably translate natural language into accurate SQL — not in a demo, but in a production banking environment with real schema complexity?

The Approach

I architected GenBI using a RAG (Retrieval-Augmented Generation) pipeline on top of LangGraph. The core idea was to give the model the right context at query time — not by stuffing the entire schema into the prompt, but by retrieving only the relevant table and column definitions using a vector database.

Two techniques drove the accuracy improvement from 30% to 70%:

Chain-of-thought prompting — rather than asking the model to produce SQL directly, I prompted it to reason through the query step-by-step: identify the relevant tables, understand the join conditions, then write the SQL. This dramatically reduced hallucinated column names and incorrect aggregations.

Persona engineering — the model was given an explicit role as a "senior data analyst at a Taiwanese bank" with specific instructions about naming conventions, business logic, and common query patterns. This grounded its outputs in the actual domain context.

Results

Query accuracy went from 30% to 70% — meaning 7 in 10 natural language questions produced correct, runnable SQL on the first attempt. Non-technical users could now interrogate transaction data directly, reducing analyst queue depth and enabling faster business decisions.

This was one of the first internal LLM deployments at the bank, shipped in early 2025.

What I'd Do Differently

The remaining 30% gap is largely schema ambiguity — cases where two columns could plausibly answer the question. A feedback loop that captures user corrections and fine-tunes retrieval ranking would be the next meaningful lever.
